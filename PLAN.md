# План: объединение листов из нескольких Excel-файлов в один

Дата: 2026-07-16. Статус: требования уточнены, решение утверждено, идёт реализация.

**Задача**: объединить первый видимый лист каждого Excel-файла из выбранной папки
в один итоговый `.xlsx`; каждый лист переименовывается по имени исходного файла.
Совместимость: Excel 2007–2024. GUI для одного пользователя (Илья).

---

## 1. Утверждённые требования

| № | Вопрос | Решение |
|---|---|---|
| 1 | Форматы источников | Все: `.xlsx`, `.xls`, `.xlsm`, `.xlsb` |
| 2 | Листов в файле | Берётся только **первый видимый** лист |
| 3 | Полнота переноса | **Полная**: форматирование, диаграммы, сводные, картинки |
| 4 | Формулы | Переносятся формулами, как есть |
| 5 | Объёмы | ~37–40 файлов, 350 КБ–1.5 МБ, ≤250 строк |
| 6 | Вход | Папка с файлами (без подпапок) |
| 7 | Ошибочный файл | Пропустить, показать в отчёте |
| 8 | Запуск | GUI, без расписания |
| 9 | Целевая машина | Python нет; Excel установлен; Windows |
| 10 | Скрытые листы | Не включать |
| 11 | Выходной файл | Имя задаётся вручную; папка сохранения выбирается; при существующем файле — вопрос о перезаписи |
| 12 | Антивирус | Kaspersky не должен триггериться |

## 2. Финальный стек: C# (.NET Framework 4.8, WinForms) + COM-автоматизация Excel

Исходный выбор Python пересмотрен и согласован с заказчиком, причины:

- **Требование №3 (переносить всё)** исключает openpyxl/excelize/exceljs: ни одна
  библиотека не переносит диаграммы, сводные таблицы и картинки. Единственный
  безпотерьный механизм — `Worksheet.Copy` самого Excel через COM.
- **Требование №1 (`.xls`, `.xlsb`)** — openpyxl их не читает; Excel через COM
  открывает всё нативно.
- **Требования №9 + №12** — PyInstaller-бандл (~60 МБ) с загрузчиком — частый
  ложный триггер антивирусов. C#-exe (~100 КБ), собранный встроенным в Windows
  `csc.exe` без упаковщиков, Kaspersky практически не трогает; .NET Framework 4.8
  уже есть на целевой машине, устанавливать ничего не нужно.
- COM-автоматизация Office — родная область .NET; поздняя привязка (`dynamic`)
  избавляет от зависимости от версии Office (2007–2024).

## 3. Архитектура

Один exe, слои разделены (SRP), GUI не знает про COM, сервис не знает про GUI:

```
src/
  Program.cs       — точка входа; GUI по умолчанию; режим --cli <папка> <файл>
                     для автотестов и скриптования
  MainForm.cs      — окно: папка-источник, имя файла, папка сохранения,
                     прогресс, живой журнал по файлам, отмена, ссылки
                     «Открыть файл/папку»; drag&drop папки
  MergeService.cs  — вся работа с Excel COM: скрытый экземпляр Excel,
                     перенос листов, отчёт; события Progress/FileDone;
                     работает в фоновом STA-потоке
  SheetNamer.cs    — имя листа из имени файла: запрещённые символы → «_»,
                     обрезка до 31 символа, дедупликация суффиксами _2, _3…
  Dialogs.cs       — нативные TaskDialog-алерты (перезапись, ошибки)
                     с фолбэком на MessageBox
  FolderPicker.cs  — современный диалог выбора папки (IFileOpenDialog)
                     с фолбэком на FolderBrowserDialog
  UserSettings.cs  — запоминание последних путей (%APPDATA%\ExcelMerger)
  AssemblyInfo.cs  — метаданные сборки (снижают подозрительность для АВ)
app.manifest       — asInvoker, DPI-aware, common controls v6
build.cmd          — сборка встроенным csc.exe (/codepage:65001, без упаковщиков)
```

Алгоритм `MergeService.Merge`:

1. Собрать файлы папки: расширения `.xlsx/.xls/.xlsm/.xlsb`, пропустить `~$*`
   и сам итоговый файл; сортировка по имени — детерминированный порядок листов.
2. Запустить скрытый Excel: `DisplayAlerts=false`, `AutomationSecurity=3`
   (макросы источников не выполняются), `AskToUpdateLinks=false`.
3. Создать итоговую книгу с одним листом-заглушкой (служебное имя зарезервировано
   в SheetNamer, чтобы не столкнуться с именем источника).
4. Для каждого файла: открыть ReadOnly, `UpdateLinks:=0`, заведомо неверный
   Password (защищённый файл даёт исключение, а не зависший диалог) →
   найти первый видимый лист → `Sheet.Copy After:=последний` → переименовать
   через SheetNamer → закрыть источник без сохранения. Любая ошибка файла →
   SKIPPED с причиной, процесс продолжается.
5. Удалить заглушку, `SaveAs(..., 51)` → `.xlsx` (родной формат Excel 2007–2024).
6. `finally`: закрыть книги, `Quit`, освободить COM, двойной GC —
   без зомби-процессов EXCEL.EXE.

## 4. Edge cases — как закрыты

| Кейс | Решение |
|---|---|
| Дубликаты имён листов (одно базовое имя с разными расширениями) | Дедупликация `_2, _3…` с учётом лимита 31 |
| Имя файла >31 символа / с `: \ / ? * [ ]` / `History` | Санитизация и обрезка в SheetNamer |
| Битый / не-Excel файл | Исключение при открытии → SKIPPED с причиной |
| Запароленный файл | Фиктивный пароль → исключение → SKIPPED «защищён паролем» |
| Нет видимых листов | SKIPPED «нет видимых листов» |
| Пустой лист | Переносится (валидный результат) |
| Файл с макросами (.xlsm) | Макросы не выполняются (AutomationSecurity); в итоговом .xlsx макросы книги не сохраняются — данные листа переносятся полностью |
| Система дат 1904 в источнике | Лист переносится, в отчёте предупреждение «проверьте даты» |
| Временные файлы Excel `~$*` | Пропускаются при сканировании |
| Итоговый файл лежит во входной папке | Исключается из источников |
| Итоговый файл существует | Нативный алерт: «Перезаписать?» |
| Итоговый файл открыт в Excel | Понятная ошибка «файл занят», без падения |
| Отмена пользователем | Останов после текущего файла, итоговый файл не создаётся |
| Зомби EXCEL.EXE | finally + Quit + FinalReleaseComObject + GC×2 |
| Kaspersky | Чистый .NET-exe без упаковщиков, метаданные версии, манифест asInvoker, без сети и записи вне выбранных папок |

Известные ограничения (согласованы): формулы со ссылками на другие книги/листы
после переноса могут ссылаться на внешние источники (заказчику не критично);
VBA-макросы книг-источников в итоговый файл не переносятся.

## 5. План реализации

1. ~~Уточнение требований~~ — готово (таблица выше).
2. Исходники по архитектуре п. 3.
3. Сборка `build.cmd` встроенным csc.
4. Тестовый корпус (генерация через Excel COM): нормальный файл
   (формат/формула/объединённые ячейки), пустой, `.xls`, `.xlsm`-дубль имени,
   скрытый первый лист, длинное имя, символы `[ ]` в имени, запароленный, битый.
5. Прогон `ExcelMerger.exe --cli testdata out.xlsx`, проверка итогового файла
   (имена листов, значения, формула, объединение).
6. Ручная проверка GUI Ильёй; README с инструкцией по сборке и развёртыванию
   (скопировать один exe).

## 6. Развёртывание

Скопировать `ExcelMerger.exe` на целевую машину — всё. Требования: Windows 10/11,
Microsoft Excel, .NET Framework 4.8 (входит в Windows 10 1903+). Установка,
права администратора, сеть — не нужны.
