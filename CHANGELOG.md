# Журнал изменений

Формат — по [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/),
версии — по [SemVer](https://semver.org/lang/ru/).

## [1.8.1] — 2026-07-17

### Добавлено
- **Масштаб миниатюр PDF**: ползунок и Ctrl+колесо мыши. Страница рендерится
  один раз, при зуме плитки пересобираются из кэша (GDI, без повторного WinRT),
  пересборка троттлится — без просадок. Юнит-тесты `ThumbZoom`.

### Изменено
- **Новые значки на карточках выбора**: документ с загнутым уголком в стиле
  file-excel (зелёный лист с таблицей для Excel, красный с «PDF») вместо
  прежней абстрактной сетки.

### Исправлено
- Плитка миниатюры больше не превышает лимит `ImageList` (256×256): на максимуме
  масштаба WinForms бросал исключение. Границы масштаба скорректированы,
  добавлен защитный кламп и регрессионный тест.

## [1.8.0] — 2026-07-17

### Изменено
- **Ребренд: iwo Helper Desktop** — новое название и иконка (logo). Название
  обновлено в заголовках окон, окне «О программе», отчётах, метаданных сборки
  и папке данных (`%APPDATA%\iwo Helper Desktop`). Внутренние инструменты —
  «Свод Excel» и «Объединение PDF».
- **Сборка переведена на dotnet SDK** (SDK-проект `iwoHelperDesktop.csproj`,
  net48): один exe `dist/iwoHelperDesktop.exe`, PdfSharp по-прежнему вшит
  ресурсом. Это открыло доступ к WinRT (Windows.Data.Pdf) для миниатюр через
  NuGet `Microsoft.Windows.SDK.Contracts` — только на этапе компиляции,
  в поставку не входит; на целевой машине ничего не ставится.

### Добавлено
- **Стартовый экран выбора инструмента**: карточки «Свод Excel» и «Объединение
  PDF» с описанием; после закрытия инструмента выбор показывается снова.
- **Миниатюры страниц PDF**: инструмент «Объединение PDF» показывает сетку
  превью реальных страниц (системный движок Windows.Data.Pdf), перестановка
  перетаскиванием миниатюр и кнопками. Рендер идёт в фоне (отдельный поток),
  недоступность движка (напр. на Windows Server) — штатный фолбэк на заглушки.
  Тесты: `verify_thumb.ps1` (рендер и соотношение сторон) и `--thumbcheck`
  (чистое завершение процесса после WinRT-рендера).

### Исправлено
- Форсированный выход процесса (`FastExit`/`ExitProcess`) после работы с WinRT:
  штатная финализация COM-обёрток Windows.Data.Pdf роняла процесс при выгрузке;
  критичная очистка (настройки, Quit COM Excel/Word) выполняется детерминированно
  до выхода.

## [1.7.0] — 2026-07-17

### Добавлено
- **Объединение PDF** (меню «Инструменты»): выбор PDF-файлов, единый список
  страниц, перестановка кнопками ▲▼ и перетаскиванием, удаление, сохранение
  в один документ. Страницы копируются без переконвертации — сканы, печати
  и подписи не искажаются (PDFsharp, MIT, вшит в exe ресурсом — по-прежнему
  один файл наружу). Битые/защищённые PDF пропускаются с причиной.
- Тесты: юнит на модель порядка страниц (перестановка/перенос/удаление),
  интеграционный `verify_pdf.ps1` (порядок и дубль страницы проверяются
  по габаритам A4/A5/альбомная), `verify_embedded.ps1` (резолв вшитого
  PdfSharp из ресурса exe в чистой папке).

## [1.6.0] — 2026-07-16

### Добавлено
- **Сопроводительная записка Word**: ссылка «Записка Word» после слияния —
  `.docx` рядом со сводом (период, счётчики, таблица пропущенных с причинами),
  оформление по ГОСТ Р 7.0.97-2016; генерация через COM установленного Word,
  чистая модель текста покрыта юнит-тестами, документ — интеграционным
  (`tests/verify_note.ps1`).
- Сортировка журнала кликом по заголовку колонки (естественное сравнение,
  повторный клик меняет направление, системная стрелка в заголовке).
- Примечание «файл содержит макросы (не выполнялись)» в журнале и оглавлении
  для источников с VBA — при сохранении свода в `.xlsm`/`.xls` код листа
  переносится вместе с листом, в `.xlsx` отбрасывается.
- Заголовок «Журнал обработки» над списком результатов.
- Интеграционный тест дослияния пропущенных (`tests/verify_retry.ps1`).

## [1.5.0] — 2026-07-16

### Добавлено
- Кнопка **«Повторить пропущенные»**: исправленные файлы дослияются в уже
  существующий свод без полного пересбора; оглавление перестраивается по
  общему итогу, порядок и успешные листы сохраняются.
- **Копирование строк журнала** — Ctrl+C или контекстное меню: строка
  «файл → лист → причина» в формате отчёта, удобно пересылать владельцу
  битого файла.
- **CHANGELOG.md** (этот файл), ссылка из README.

### Изменено
- Опция «Заменить формулы значениями» больше **не запоминается** между
  запусками: режим меняет содержимое свода и включается осознанно каждый раз.

## [1.4.0] — 2026-07-16

### Добавлено
- **Выбор формата итогового файла**: `.xlsx`, `.xlsm`, `.xlsb`, `.xls`
  (выпадающий список; в CLI формат определяется расширением пути).
- Интеграционный прогон в `.xlsb` в общем наборе тестов.

### Изменено
- Чекбоксы в фирменном стиле: белая галочка на зелёном фоне, hover,
  фокус-кольцо (`AccentCheckBox`).
- Колонки журнала делят ширину окна пропорционально.
- По гайдлайнам Windows убраны многоточия у пунктов «Как пользоваться»
  и «О программе»; в справке поправлена пунктуация.

## [1.3.0] — 2026-07-16

## [1.2.0] — 2026-07-16

### Добавлено
- **Прогресс на кнопке панели задач** (ITaskbarList3) и мигание окна
  по завершении, если пользователь работает в другом приложении.
- **Ранняя проверка блокировки** итогового файла: занятый файл
  обнаруживается до запуска Excel, а не после обработки всех источников.
- **История отчётов** в `%APPDATA%\ExcelMerger\reports` (не более трёх),
  ссылка «Открыть отчёт» после слияния.
- CI (GitHub Actions): сборка, юнит-тесты, смоук GUI; публикация exe
  в Releases по тегу `v*`.
- `tools/sign.ps1` — подпись exe самоподписанным сертификатом (SHA256).
- `tests/run_all` — вся пирамида тестов одной командой.

## [1.1.1] — 2026-07-16

### Исправлено
- **Экранирование строк при записи в ячейки**: имя файла или строковый
  результат формулы, начинающийся с «=», превращался в формулу (инъекция);
  ведущий апостроф строки терялся. Проверено экспериментально, покрыто
  юнит- и интеграционными тестами.

## [1.1.0] — 2026-07-16

### Добавлено
- **Лист «Содержание»**: оглавление свода с гиперссылками на листы
  и статусами всех файлов, включая пропущенные (опция, по умолчанию вкл.).
- **Естественный порядок файлов** как в Проводнике: «Отчет 2» раньше
  «Отчет 10» (StrCmpLogicalW).
- Опция **«Заменить формулы значениями»** — свод без внешних ссылок;
  объединённые ячейки обрабатываются поячеечным фолбэком.
- OLE message filter: авто-повтор COM-вызовов, отклонённых занятым Excel.
- Ручной режим пересчёта на время слияния (быстрее на формулах).
- Юнит-тесты без внешних фреймворков (`tests/build_tests.cmd`).

## [1.0.0] — 2026-07-16

Первый выпуск.

- Объединение первого видимого листа каждого Excel-файла папки в один
  `.xlsx` через COM установленного Excel — без потери форматирования,
  формул, объединённых ячеек, диаграмм и сводных таблиц.
- Форматы источников: `.xlsx`, `.xls`, `.xlsm`, `.xlsb`; битые и
  запароленные файлы пропускаются с причиной; скрытые листы не переносятся;
  имена листов — из имён файлов с дедупликацией и лимитом 31 символ.
- WinForms GUI: живая валидация, ход обработки, журнал с расцветкой,
  drag&drop папки, запоминание путей; иконка и фирменный стиль.
- CLI-режим `--cli` для скриптов и автотестов; интеграционные тесты
  на корпусе из 13 файлов; один exe ~65 КБ без зависимостей
  (.NET Framework 4.8, компилятор из состава Windows).
