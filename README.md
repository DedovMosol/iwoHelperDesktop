# iwo Helper Desktop

![CI](https://github.com/DedovMosol/iwoHelperDesktop/actions/workflows/ci.yml/badge.svg)

История версий — в [CHANGELOG.md](CHANGELOG.md).

Набор офисных инструментов в одном приложении:

- **Свод Excel** — объединение первого видимого листа каждого Excel-файла папки
  в один сводный файл (`.xlsx`/`.xlsm`/`.xlsb`/`.xls`) без потери оформления
  (форматирование, формулы, диаграммы, сводные), с оглавлением, заменой формул
  значениями и сопроводительной запиской Word.
- **Объединение PDF** — сборка одного PDF из нескольких: выбор нужных страниц
  и их порядка, без переконвертации.

При запуске программа предлагает выбрать инструмент.

## Требования

- Windows 10/11 (нужен .NET Framework 4.8 — входит в Windows 10 1903+);
- установленный Microsoft Excel (2007–2024); для записки — Microsoft Word;
- права администратора и сеть **не нужны**.

## Структура репозитория

```
src/     исходники приложения          tools/   подпись exe
tests/   юнит- и интеграционные тесты  build/   входы сборки (манифест, иконки, PdfSharp.dll)
dist/    результат сборки (не в git)
```

Сборка через `dotnet` SDK. Единственная поставляемая зависимость —
`build/PdfSharp.dll` (MIT); она вшивается в exe ресурсом, наружу — один файл.
Для миниатюр PDF используется системный `Windows.Data.Pdf` (WinRT); проекции
для компиляции берутся из NuGet `Microsoft.Windows.SDK.Contracts` (только сборка,
в поставку не входит).

## Развёртывание

Скопировать один файл `iwoHelperDesktop.exe` на целевую машину — из
[Releases](https://github.com/DedovMosol/iwoHelperDesktop/releases) или собранный
локально (`dist\iwoHelperDesktop.exe`). Всё.

Программа не упакована и не обфусцирована (обычная .NET-сборка, ~0,7 МБ —
включает вшитый ресурсом PdfSharp), не обращается к сети, пишет только
в выбранные пользователем папки и в `%APPDATA%\iwo Helper Desktop` (настройки
и история отчётов) — антивирусам триггериться не на что.

## Использование (GUI)

При запуске программа предлагает выбрать инструмент: **«Свод Excel»**
(объединение листов Excel) или **«Объединение PDF»**. После закрытия
инструмента снова показывается выбор — можно перейти к другому без перезапуска.

### Свод Excel

1. Указать папку с исходными файлами (кнопка «Обзор…» или перетащить папку
   в окно). Программа сразу показывает, сколько файлов Excel найдено.
2. Задать имя сводного файла и его формат (`.xlsx` / `.xlsm` / `.xlsb` / `.xls`).
   В параметрах — «Листы»: брать из каждого файла только первый лист или все.
3. При необходимости сменить папку сохранения (по умолчанию — папка с исходными).
4. Нажать «Объединить». Ход — в журнале обработки и на кнопке панели задач;
   по завершении в неактивном окне кнопка мигает.
5. Если сводный файл уже существует — программа спросит о перезаписи;
   занятый файл (открыт в Excel) обнаруживается сразу, до начала обработки.

**Журнал обработки** — построчный результат по каждому файлу: имя листа в своде,
статус, причина пропуска или предупреждение (например, «файл содержит макросы»).
Колонки сортируются кликом по заголовку, строки копируются в буфер по Ctrl+C
или через контекстное меню — удобно пересылать причину владельцу файла.

После слияния доступны: «Открыть файл», «Открыть папку», «Открыть отчёт»
(текстовая история хранится в `%APPDATA%\iwo Helper Desktop\reports`, не более трёх
последних; доступ также через меню Справка → «Папка отчётов») и
**«Записка Word»** — сопроводительная записка `.docx` рядом со сводом:
период, счётчики, таблица пропущенных с причинами, оформление по
ГОСТ Р 7.0.97-2016 (поля 30/15/20/20 мм, Times New Roman 14, красная строка,
полуторный интервал).

Если часть файлов пропущена, появляется кнопка **«Повторить пропущенные»** —
исправленные файлы дослияются в существующий свод без полного пересбора
(оглавление перестраивается, порядок и готовые листы не затрагиваются).

Поддерживаемые исходные форматы: `.xlsx`, `.xls`, `.xlsm`, `.xlsb`.
Листы в своде идут в естественном порядке имён файлов — как в Проводнике:
«Отчет 2» раньше «Отчет 10».

Параметры (формат и «Содержание» запоминаются; «Заменить формулы значениями»
при каждом запуске выключен — режим включается осознанно):

- **Лист «Содержание»** (включено по умолчанию) — первым листом свода создаётся
  оглавление: гиперссылки на каждый лист и статусы всех файлов, включая
  пропущенные с причинами. Шапка оглавления закреплена.
- **Заменить формулы значениями** — свод перестаёт зависеть от исходных файлов:
  вместо формул сохраняются вычисленные значения (форматирование и объединённые
  ячейки не затрагиваются).

Справка — в меню (F1): «Как пользоваться», «Папка отчётов», «О программе».

## Объединение PDF

Инструмент **«Объединение PDF»** со стартового экрана: отдельное окно для
склейки. Выбираешь документы (кнопкой или перетаскиванием), получаешь **сетку
миниатюр** страниц, переставляешь их перетаскиванием или кнопками, лишние
удаляешь, сохраняешь в один PDF. Страницы копируются **как есть**, без
переконвертации — сканы, печати и подписи не искажаются. Битые и защищённые
паролем PDF пропускаются с понятной причиной.

Склейка — на PdfSharp (MIT, вшит в exe ресурсом). Миниатюры рисует системный
движок Windows.Data.Pdf; если он недоступен (например, на Windows Server) —
показываются заглушки, инструмент продолжает работать. Наружу поставляется
один файл, ничего не устанавливается.

## Поведение в особых случаях

- битый / запароленный / нечитаемый файл — пропускается, причина в журнале,
  оглавлении и отчёте;
- скрытые листы не переносятся (берётся первый **видимый** лист);
- совпадающие имена (например, `Отчет.xls` и `Отчет.xlsx`) — второй лист
  получает суффикс `_2`;
- имя длиннее 31 символа обрезается; символы `: \ / ? * [ ]` заменяются на `_`;
- временные файлы Excel (`~$…`) игнорируются;
- макросы источников **не выполняются**; файлы с VBA помечаются примечанием.
  В свод `.xlsx` код листа не попадает (Excel отбрасывает), в `.xlsm`/`.xls`
  переносится вместе с листом;
- если ни один лист перенести не удалось или объединение отменено — сводный
  файл не создаётся (а при повторе пропущенных — не изменяется).

## Режим командной строки (для тестов и скриптов)

```
iwoHelperDesktop.exe --cli <папка_с_исходными> <путь_к_своду> [--toc] [--values]
```

Формат свода определяется расширением пути (`.xlsx`/`.xlsm`/`.xlsb`/`.xls`).
`--toc` — добавить лист «Содержание», `--values` — заменить формулы значениями,
`--allsheets` — брать все видимые листы (иначе только первый). В CLI опции
по умолчанию выключены и включаются явно.
Отчёт пишется в `<свод>.report.txt`. Коды выхода: `0` — все файлы перенесены,
`2` — есть пропущенные, `1` — ошибка (сводный файл не создан).

## Сборка из исходников

```
build.cmd
```

Требуется `dotnet` SDK (6+); собирает SDK-проект `iwoHelperDesktop.csproj`
(целевой фреймворк .NET Framework 4.8). Результат — `dist\iwoHelperDesktop.exe`,
один файл. На целевой машине ничего не устанавливается (нужен лишь
.NET Framework 4.8, входит в Windows 10).

Иконка приложения — `build/app.ico` (многоразмерная, из `build/logo.ico`).
Палитра интерфейса — `src/Theme.cs`.

Подпись exe (опциональный шаг перед развёртыванием):

```
powershell -NoProfile -File tools\sign.ps1
```

Создаёт (один раз) самоподписанный сертификат в `Cert:\CurrentUser\My`
и подписывает exe (SHA256, метка времени при доступной сети). Даёт целостность
файла и постоянного издателя; «доверенной» для Windows подпись станет только
после добавления сертификата в доверенные корневые на целевой машине.

## CI

GitHub Actions (`.github/workflows/ci.yml`): на каждый пуш — сборка,
юнит-тесты, смоук-тест GUI, артефакт `iwoHelperDesktop.exe`; на тег `v*` — публикация
exe в Releases. Интеграционные тесты требуют Office и выполняются только локально.

## Тесты

Вся пирамида одним запуском (нужны установленные Excel и Word):

```
tests\run_all.cmd
```

Последовательность: сборка → юнит-тесты → смоук GUI → генерация корпуса →
базовый прогон → прогон с оглавлением и заменой формул → прогон в `.xlsb` →
дослияние пропущенных → вшитый PdfSharp → объединение PDF → миниатюры PDF →
записка Word → контроль зомби-процессов Excel.

По отдельности — юнит-тесты (без Office и внешних фреймворков, свой мини-раннер):

```
tests\build_tests.cmd    # сборка и запуск; код выхода 0 = все прошли
```

Интеграционные проверки: `tests\verify.ps1` (базовое поведение),
`verify_toc.ps1` (оглавление и замена формул), `verify_format.ps1` (свод
в `.xlsb`), `verify_retry.ps1` (дослияние пропущенных), `verify_pdf.ps1`
(объединение PDF), `verify_thumb.ps1` (рендер миниатюр WinRT),
`verify_note.ps1` (записка Word, включая поля и шрифт по ГОСТ).

Корпус покрывает: форматирование/формулы/объединённые ячейки (в том числе формулу
внутри объединённой ячейки и формулы со строковыми результатами), пустой файл,
`.xls`/`.xlsm`/`.xlsb`, дубликат базового имени, скрытый первый лист, имя длиннее
31 символа, скобки в имени файла, естественный порядок («Отчет 2» раньше
«Отчет 10»), запароленный файл, битый файл, временный `~$`-файл.

## Важно для сопровождения кода

Office COM используется через позднее связывание (`dynamic`) — нет зависимости
от версии Office. Правило: **никогда не выполнять динамических операций над
закрытым COM-объектом** (сохранять `dynamic`-ссылку в `object`-переменную до
`Close`/`Quit`) — динамическая привязка на мёртвом объекте роняет процесс с
`COMException 0x80010114` до входа в метод, мимо любых try/catch (подробности —
комментарий у `ComSafe.Release`). Текст в ячейки Excel писать только
через `CellText.EscapeValues` — строка «=x» иначе превращается в формулу.
