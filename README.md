# Свод листов Excel (ExcelMerger)

Объединяет **первый видимый лист** каждого Excel-файла из выбранной папки в один
итоговый `.xlsx`. Лист получает имя исходного файла. Перенос без потерь — средствами
самого Excel (COM): форматирование, формулы, объединённые ячейки, диаграммы,
сводные таблицы, картинки.

## Требования

- Windows 10/11 (нужен .NET Framework 4.8 — входит в Windows 10 1903+);
- установленный Microsoft Excel (2007–2024);
- права администратора и сеть **не нужны**.

## Развёртывание

Скопировать один файл `ExcelMerger.exe` на целевую машину. Всё.

Программа не упакована и не обфусцирована (обычная .NET-сборка, ~40 КБ),
не обращается к сети, пишет только в выбранные пользователем папки и в
`%APPDATA%\ExcelMerger\settings.txt` (запоминает последние пути) — антивирусам
триггериться не на что.

## Использование (GUI)

1. Указать папку с исходными файлами (кнопка «Обзор…» или перетащить папку в окно).
2. Задать имя итогового файла (расширение `.xlsx` добавится само).
3. При необходимости сменить папку сохранения (по умолчанию — папка с исходными).
4. Нажать «Объединить». Ход обработки и результат по каждому файлу — в списке.
5. Если итоговый файл уже существует — программа спросит о перезаписи.

Поддерживаемые исходные форматы: `.xlsx`, `.xls`, `.xlsm`, `.xlsb`.
Листы в итоговой книге идут в естественном порядке имён файлов —
как в Проводнике: «Отчет 2» раньше «Отчет 10».

Параметры (запоминаются между запусками):

- **Лист «Содержание»** (включено по умолчанию) — первым листом свода создаётся
  оглавление: гиперссылки на каждый лист и статусы всех файлов, включая
  пропущенные с причинами. Шапка оглавления закреплена.
- **Заменить формулы значениями** — свод перестаёт зависеть от исходных файлов:
  вместо формул сохраняются вычисленные значения (форматирование и объединённые
  ячейки не затрагиваются).

## Поведение в особых случаях

- битый / запароленный / нечитаемый файл — пропускается, причина в списке;
- скрытые листы не переносятся (берётся первый **видимый** лист);
- совпадающие имена (например, `Отчет.xls` и `Отчет.xlsx`) — второй лист
  получает суффикс `_2`;
- имя длиннее 31 символа обрезается; символы `: \ / ? * [ ]` заменяются на `_`;
- временные файлы Excel (`~$…`) игнорируются;
- макросы исходных `.xlsm` не выполняются и в итоговый `.xlsx` не переносятся;
- если ни один лист перенести не удалось или объединение отменено — итоговый
  файл не создаётся.

## Режим командной строки (для тестов и скриптов)

```
ExcelMerger.exe --cli <папка_с_исходными> <путь_к_итоговому.xlsx> [--toc] [--values]
```

`--toc` — добавить лист «Содержание», `--values` — заменить формулы значениями
(в CLI обе опции по умолчанию выключены и включаются явно).
Отчёт пишется в `<итоговый>.report.txt`. Коды выхода: `0` — все файлы перенесены,
`2` — есть пропущенные, `1` — ошибка (итоговый файл не создан).

## Сборка из исходников

```
build.cmd
```

Используется компилятор C#, встроенный в Windows
(`%WINDIR%\Microsoft.NET\Framework64\v4.0.30319\csc.exe`, язык C# 5).
Внешние зависимости отсутствуют.

Иконка приложения: дизайн — `assets/icon.svg`, встраиваемый `app.ico`
генерируется из того же дизайна командой `tools\make_icon.cmd`
(перегенерация нужна только при изменении дизайна, `app.ico` закоммичен).
Палитра интерфейса и иконки — `src/Theme.cs`.

## Тесты

Юнит-тесты (без Excel и внешних фреймворков — собственный мини-раннер):

```
tests\build_tests.cmd    # сборка и запуск; код выхода 0 = все прошли
```

Интеграционные (нужен установленный Excel):

```
powershell -NoProfile -File tests\make_testdata.ps1   # генерирует корпус
ExcelMerger.exe --cli tests\testdata tests\out\Свод.xlsx
powershell -NoProfile -File tests\verify.ps1          # базовое поведение
ExcelMerger.exe --cli tests\testdata tests\out\Свод_toc.xlsx --toc --values
powershell -NoProfile -File tests\verify_toc.ps1      # оглавление и замена формул
```

Корпус покрывает: форматирование/формулы/объединённые ячейки (в том числе формулу
внутри объединённой ячейки), пустой файл, `.xls`/`.xlsm`/`.xlsb`, дубликат базового
имени, скрытый первый лист, имя длиннее 31 символа, скобки в имени файла,
естественный порядок («Отчет 2» раньше «Отчет 10»), запароленный файл, битый файл,
временный `~$`-файл.

## Важно для сопровождения кода

Excel COM используется через позднее связывание (`dynamic`) — нет зависимости от
версии Office. Правило: **никогда не передавать `dynamic`-переменную с закрытым
COM-объектом в обычный метод без приведения `(object)`** — динамическая привязка
на мёртвом объекте роняет процесс с `COMException 0x80010114` до входа в метод,
мимо любых try/catch (подробности — комментарий у `MergeService.ReleaseCom`).
